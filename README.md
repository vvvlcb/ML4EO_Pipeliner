# ML4EO Pipeliner User Manual

ML4EO Pipeliner is a web platform for building, running, viewing, and sharing Earth observation workflows without requiring users to write code for every data acquisition, preprocessing, inference, and visualization step.

Open the tool here: [https://pipeliner.ml4eo.world](https://pipeliner.ml4eo.world)

[Slide manual](https://github.com/vvvlcb/ML4EO_Pipeliner/blob/main/ML4EO_Pipeliner_Demo_v1.pptx)

## What Pipeliner Does

Pipeliner is designed to reduce the gap between Earth observation research code and usable operational tools.

It helps users:

- Acquire and process Earth observation imagery without writing boilerplate code.
- Build time-series datasets over a selected area of interest.
- Run source, index, and model tools online.
- View output layers on a map.
- Compare results over time.
- Download outputs such as GeoTIFF, CSV, SVG, PNG, and STAC-style collection exports where available.
- Share results through public timespaces or collaborative workspaces.

The platform is especially useful when non-technical collaborators need to inspect EO results, compare time periods, or evaluate model outputs without running notebooks or local machine learning environments.

## Access Options

You can use Pipeliner in two ways.

| Access path | Best for | What you can do |
| --- | --- | --- |
| Public demo | Fast exploration and learning | View public AOIs, timespaces, slices, layers, and analysis outputs in read-only mode |
| Registered account | Creating and managing your own work | Create shareable workspaces, fetch/process imagery, manage datasets, run tools, publish timespaces, and export datasets |

To register or sign in:

1. Open [https://pipeliner.ml4eo.world](https://pipeliner.ml4eo.world).
2. Click **Sign in**.
3. Continue with Google, or use email and password.
4. To create a new account, choose **Create an account**.
5. Enter an email address, password, and optional display name.
6. Use the emailed 6-digit verification code to confirm your address.

## Core Concepts

### AOI

An **AOI**, or Area of Interest, is the geographic area you want to study. It may represent a city, river corridor, fire perimeter, agricultural area, coastline, or any other spatial region.

### Timespace

A **Timespace** combines an AOI with one or more time slices. In the manual, this is described as similar to a STAC Collection: it organizes imagery and outputs for a location over time.

### Slice

A **Slice** is one time window inside a timespace. For example, a monthly slice may cover `2024-11-01` to `2024-12-01`.

### Tool

A **Tool** performs a data operation on selected slices. Tools may fetch source imagery, calculate indices, or run model inference.

Tool output types include:

- **Source**: Original imagery from Earth observation data providers.
- **Index**: Numerical products computed from source imagery, such as NDVI or change layers.
- **Model**: Evaluated imagery generated by machine learning models.

### Layer

A **Layer** is a map-viewable output, such as an RGB preview, a source band, an index result, or a model result. Layers can be loaded onto the map and compared visually.

## Interface Overview

The main Pipeliner interface contains these areas.

| Area | Purpose |
| --- | --- |
| Left panel | Switch between Public, Workspace, and Layers panels |
| Map | View AOIs, slices, loaded imagery, and model outputs |
| Coordinate bar | Show and copy map coordinates, switch projection, or go to a coordinate |
| Slice panel | Inspect available layers and slice-level outputs |
| Analysis panel | View charts, statistics, comparisons, and export tables/figures |
| Timeline | Move between slices and inspect outputs across time |

## Public Demo Workflow

Use the public demo first if you want to learn the interface before creating an account.

1. Open [https://pipeliner.ml4eo.world](https://pipeliner.ml4eo.world).
2. Stay in the **Public** panel.
3. Use the search box or workspace filter to find a public case.
4. Select an AOI.
5. Select a timespace under that AOI.
6. Use the map, timeline, Slice panel, and Analysis panel to inspect results.

Public demo content is read-only. You can view and download available outputs, but you cannot modify the public workspace.

## Example-1: Gaza Conflict Demo

The manual includes a Gaza conflict demo to show time-series visualization and analysis tools.

Use it to inspect satellite-derived changes from September 2023 to January 2025.

1. Open the **Public** panel.
2. Filter to **Public Demo**, if needed.
3. Select the **Gaza** AOI.
4. Open **2023-2025 Gaza Conflicts**.
5. Use the **Slice** tab to inspect available layers.
6. Load useful layers such as:
   - RGB composite
   - NDVI
   - LULC
   - Change layer
   - Sentinel-2 source bands
   - Dynamic World class probabilities
7. Use the **Analysis** tab to inspect land-cover composition, NDVI trends, and comparison outputs.
8. Single-click two records in an analysis chart to compare them.
9. Double-click one record to open that slice.

Available public outputs observed in the live Gaza demo include:

- Sentinel-2 Surface Reflectance Harmonized source data.
- Dynamic World land-cover class probabilities.
- RGB composite preview.
- NDVI index output.
- LULC model output.
- Change result for selected date ranges.
- GeoTIFF downloads for individual output layers.
- CSV and SVG downloads from the analysis view.

## Layer Controls

Open the **Layers** panel to control map display.

Reference layers include:

- OpenStreetMap
- Labels overlay
- Esri World Imagery

For each reference layer, you can toggle visibility and adjust opacity.

Image layers include step results and GeoTIFFs. Outputs can appear here after you load a slice result, run a tool, or drop a GeoTIFF with an embedded bounding box onto the map.

Common layer tasks:

| Task | How to do it |
| --- | --- |
| Show a source or result layer | Open **Slice**, find the layer, and click **Load** |
| Temporarily preview a layer | Click the layer tab or row in the Slice panel |
| Compare multiple layers | Load them into **Image Layers**, then adjust opacity/order |
| Save the current map | Click **Save current view (PNG)** in the Layers panel |
| Download a raster output | Click **TIF** or **Download slice (TIF)** where available |

## Coordinate and Projection Tools

The coordinate bar shows the current longitude and latitude. Click a coordinate value to copy it.

Use the projection dropdown to change the displayed coordinate reference system. The live interface includes options such as:

- WGS 84 / EPSG:4326
- Web Mercator / EPSG:3857
- Equal-area world projections
- Polar projections
- National grids
- UTM zones

Use **Go to coordinate** to move the map to a known location.

## Analysis Panel

Use the **Analysis** tab when result layers support time-sequential comparison.

Depending on the selected result, Pipeliner may show:

- Land-cover composition over time.
- NDVI trends.
- Source availability summaries.
- First-last comparison outputs.
- Pixel counts and percentages.
- Chart exports.

Observed analysis exports include:

- **CSV** for table/statistical data.
- **Download SVG** for charts.
- **Fullscreen** chart viewing.

Comparison mode:

1. Open the **Analysis** tab.
2. Select a result such as LULC, NDVI, Source, or a Change comparison.
3. Single-click two records in the graph or timeline.
4. Pipeliner performs a first-last comparison and shows the difference statistically and visually.

## Guided Mode

Guided mode is for fast, predefined workflows. It is the recommended path when you want a simple task flow and do not need fine-grained control.

The manual describes Guided mode as a three-step workflow:

1. **AOI**: Select or provide the area of interest.
2. **TIME**: Choose the time range or target date.
3. **GET**: Run the workflow and generate the output.

Guided results are still stored in a timespace, so they can be inspected later in the same map, slice, layer, and analysis tools.

Example guided workflows from the manual include:

- **Vegetation Over Time**
- **Wildfire Burn Scar**

## Example-2: Eaton Fire, Altadena, Los Angeles

The manual uses the January 2025 Eaton Fire in Los Angeles County as a guided-mode example.

The recommended workflow is:

1. Choose **Guided** mode.
2. Select **Vegetation Over Time**.
3. Use the predefined AOI: **Eaton Fire - Altadena, LA**.
4. Use the date range:
   - Start: `2024-11-01`
   - End: `2025-03-01`
5. Run the workflow.
6. Inspect the NDVI time series.
7. Adjust the change threshold. The manual suggests an optimal value of about `0.15` for this case.
8. Compare monthly changes from left to right to find abnormal change.
9. Run **Wildfire Burn Scar** for the best post-event date window.
10. Evaluate the output and note limitations.

Date note: one slide in the uploaded manual lists `2026/02/01` for the wildfire burn scar date, but the Eaton Fire event timeline and the public demo visible in the live app indicate a 2025 post-event window, including `2025-02-01`. For this case, treat February 2025 as the intended analysis period unless the project team confirms otherwise.

## Pro Mode

Pro mode gives finer control over dataset construction and tool execution.

The manual describes Pro mode as three main steps:

1. **AOI**
2. **Timespace**
3. **Tool Run**

Use Pro mode when you want to build your own dataset, run specific source/index/model operations, publish a timespace, or export a STAC-style collection.

### Step 1: Create an AOI

In Pro mode, select one drawing tool:

- **Free Polygon**
- **Drag Rectangle**
- **Central Rectangle**, which is described as mobile friendly

Then:

1. Draw the AOI on the map.
2. Register it with a clear name.
3. Continue to Timespace creation.

AOI size note: the manual states that every timespace has a default AOI size limit of `150 km2`. This can be changed in workspace settings up to `2000 km2`, but large areas should be used cautiously because they increase data volume and processing cost.

### Step 2: Define a Timespace

Create a timespace by combining an AOI with time slices.

1. Give the timespace a name.
2. Create one or more time slices.
3. Stage the slices.
4. Commit the timespace.
5. Continue to Tool Run.

From the manual, a timespace can also be:

- Published so other users can access it in Public.
- Exported as a STAC Collection ZIP for downstream analysis.

### Step 3: Run Tools

Tool Run fetches source data and generates imagery layers.

Tool categories:

| Category | Meaning |
| --- | --- |
| Source | Original imagery from EO data providers |
| Index | Computed products such as NDVI or change indices |
| Model | ML model outputs evaluated from source imagery |

Before a tool runs, Pipeliner checks data availability for each slice. Availability dots represent available observations. A slice marked red means there is no data for that period.

If a slice has no data:

- Try a different date range.
- Use a wider time slice.
- Try a different source/tool.
- Check whether the AOI is too small, too cloudy, or outside the source coverage.

## Example-3: Creating Your Own STAC Dataset

Use Pro mode when your goal is to build a structured dataset for later analysis or model development.

Recommended workflow:

1. Sign in.
2. Create or select a workspace.
3. Draw and register an AOI.
4. Create a timespace.
5. Add time slices.
6. Run source tools to fetch imagery.
7. Run index tools or model tools as needed.
8. Inspect slices and layers.
9. Use the Analysis panel for quality checks.
10. Export the timespace as a STAC Collection ZIP if available.

This workflow is useful for moving from raw imagery to a reusable, standards-aligned asset catalog.

## Downloads and Exports

Depending on the panel and output type, Pipeliner supports several export paths.

| Export | Where it appears | Use |
| --- | --- | --- |
| GeoTIFF / TIF | Slice layers and tool results | Raster analysis in GIS or Python |
| Download slice (TIF) | Slice panel | Export the selected slice raster |
| CSV | Analysis panel | Export tabular statistics |
| SVG | Analysis chart | Export chart graphics |
| PNG | Layers panel | Save current map view |
| STAC Collection ZIP | Timespace export, where available | Downstream catalog use and dataset sharing |

When exporting model or index outputs, also record the AOI, time slice, source imagery, model/tool name, and date of download so results can be reproduced.

## Troubleshooting

### I cannot edit a public demo

Public demos are read-only. Sign in and use a workspace if you need to create AOIs, run tools, or manage datasets.

### A layer does not appear on the map

Check these items:

- Click **Load** next to the layer.
- Open **Layers** and confirm the image layer is visible.
- Increase layer opacity.
- Move the layer above other image layers if ordering controls are available.
- Use **Fit** or zoom to the AOI.
- Confirm the selected slice has data.

### A slice is red or has no observations

That period has no available data for the selected source/tool. Try a wider time window, another source, or a different date range.

### Analysis charts are empty

The selected result may not support time-series analysis, or the tool may not have generated outputs for the selected slices. Try another result type, such as NDVI, LULC, Source, or Change.

### The AOI is too large

The manual states a default AOI size limit of `150 km2` per timespace. Workspace settings may allow up to `2000 km2`, but large AOIs should be split into smaller regions when possible.
